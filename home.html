Here's what's actually happening: If your website uses the browser's "Use My Location" feature, the browser estimates the person's location using a combination of: GPS (on phones, if allowed) WiFi network locations Cell towers IP address If GPS isn't available or isn't accurate, it falls back to IP or WiFi positioning, which can easily be hundreds of yards or even several miles off. Different devices, browsers, and networks will return different locations. That's why one person may appear 0.4 miles away while another appears 6 miles away. The problem is probably not with your volunteer's pin. If you stored his latitude/longitude correctly, his pin isn't moving. The problem is likely the searcher's location. The best fix Instead of automatically searching from "Current Location," let users choose: Search by: 📍 Use My Current Location 🏠 Enter My Address or ZIP Code When someone enters an address, use a geocoding service (Google Maps, Mapbox, OpenStreetMap/Nominatim, etc.) to convert that address into exact coordinates. Those coordinates are usually much more accurate than browser geolocation. Many sites (like Home Depot, Walmart, and Lowe's) do exactly this because browser geolocation isn't reliable enough.
